## 4. 데이터베이스 구성 및 역할

| DB             | 역할                                                         | 비고                           |
| -------------- | ------------------------------------------------------------ | ------------------------------ |
| **PostgreSQL** | 메인 RDB — 사용자, 콘티, 찬양, 팀, 히스토리, 커뮤니티 데이터 | Supabase 호스팅                |
| **MySQL**      | PostgreSQL 대안 — 경량 서비스에 고려                         | Railway 지원                   |
| **SQLite**     | 로컬 개발/테스트 환경 전용                                   | 운영 환경엔 부적합             |
| **Redis**      | 세션 캐싱, 로그인 토큰 관리, 일시적 데이터                   | 속도 중심                      |

### 악보 저장 방식

- **PDF/이미지 악보**는 용량이 크기 때문에 Railway 단독 배포로는 저장 불가
- **Supabase Storage** 사용
- DB에는 파일 URL만 저장, 실제 파일은 클라우드 스토리지에 보관

### 로그인 설계

- 일반 이메일/비밀번호 + 소셜 로그인(Google 등) 고려
- Redis로 JWT Refresh Token 관리

## 7. .DS_Store가 git status에 뜨는 이유

`.DS_Store`를 `.gitignore`에 추가했더라도 **이미 git이 추적하고 있는 파일**이라면 무시되지 않는다.

### 해결 방법

```bash
# git 캐시에서 제거
git rm --cached .DS_Store
git rm --cached -r **/.DS_Store

# 이후 커밋
git add .gitignore
git commit -m "remove .DS_Store from tracking"
```

### 전역 gitignore 설정 (맥 환경 권장)

```bash
echo ".DS_Store" >> ~/.gitignore_global
git config --global core.excludesfile ~/.gitignore_global
```

- APK 직접 배포도 가능 (테스트 단계)

## 9. 악보 저작권 처리 방안

- 악보는 **로그인 세션 내 개인 사용**으로 제한
- 다른 유저와의 직접 악보 공유 기능은 비활성화
- **PDF 출력 시에만 악보 포함** → 출력물은 개인 오프라인 사용 목적
- 팀 내 공유 시에는 악보 파일 자체가 아닌 **콘티 정보(곡명, 키, 템포 등)만 공유**
- 추후 정식 저작권 라이선스 취득 또는 공개 저작권 악보만 허용하는 정책 검토

## 10. 팀 초대 기능 구동 방식

### 흐름

1. 팀장이 **초대 링크 또는 QR 코드 생성** (서버에서 토큰 발급, 만료 시간 설정)
2. 초대받은 유저가 링크/QR 접속 → 로그인 후 팀 자동 가입
3. 토큰은 1회용 또는 시간 제한 (예: 24시간 유효)

### 기술 구현

- 서버에서 `invite_token` 발급 (UUID or JWT)
- Redis에 토큰 저장 (TTL 설정으로 만료 관리)
- QR 코드는 `qrcode` 라이브러리로 생성
- 링크 예시: `https://worshiplog.com/invite?token=xxxx`

## 12. 커뮤니티 & 말씀 랜덤 추천 묵상 공유

### 커뮤니티

- 로그인한 사용자만 접근 가능
- 게시글/댓글 기반 소통 공간
- 콘티, 찬양 후기 등 공유 가능

### 말씀 랜덤 추천 & 묵상 공유

- 성경 말씀은 **JSON 파일로 서버에 번들 저장**
  - 위치: `server/src/common/data/bible.json` (약 7MB, 66권 전체 수록)
  - 구조: `[{ "book": "창", "chapter": 1, "verse": 1, "content": "..." }, ...]`
  - 서버에서 랜덤 추출 후 API로 클라이언트에 전달
- 매일 또는 접속 시 랜덤 말씀 추천
- 해당 말씀에 대한 묵상 내용을 커뮤니티에 공유 가능
- 말씀 → 관련 찬양 큐레이션으로 연결

```json
// 말씀 JSON 예시
{
    "book": "시편",
    "chapter": 23,
    "verse": 1,
    "content": "여호와는 나의 목자시니 내게 부족함이 없으리로다"
}
```
