## 웹 (client / server)

### 1. 팀스페이스 실시간 반영 (폴링 방식)

- SSE(Server-Sent Events) 대신 인터벌 폴링으로 교체 (안정성 이슈)
  - 현재 채팅방 메시지: **3초** 간격 폴링
  - 조직/방 목록(미읽음 카운트): **10초** 간격 폴링
- 메시지 중복 방지: 마지막 메시지 ID 비교로 불필요한 리렌더링 차단
- 서버에 org 레벨 SSE 엔드포인트(`GET /rooms/stream/org`) 추가 (향후 전환 대비 유지)
- 읽지 않은 메시지 수 `localStorage` 기반으로 표시

### 2. 콘티 복제 기능 (`내 콘티로 복제`)

- 타인의 콘티(팀 공유 콘티 등)를 내 콘티 목록으로 복제하는 기능 추가
- 복제 시 곡 목록(key, tempo, note) 그대로 복사, 악보 파일은 제외(원본 귀속)
- **본인 콘티에서는 복제 버튼 숨김** (`conti.createdBy === myUserId`)
- API: `POST /contis/:id/clone`

### 3. PDF 출력 레이아웃 개선

- `@page { margin: 0; size: A4; }` 적용 → 브라우저 인쇄 시 상단 시각/하단 URL 헤더 제거
- 여백을 `padding`으로 직접 보정 (`8mm`)
- **헤더 테이블 구조 개선**
  - 기존: `번호 | 제목 | 팀이름(별도 컬럼) | Key | 키값 | tempo | 템포값` (7컬럼)
  - 변경: `번호 | 제목 - 팀이름(병합) | Key | 키값 | tempo | 템포값` (6컬럼)
  - 번호 컬럼 폭 `44px → 68px` 통일 (`1번`, `2번 (2p)` 모두 안 잘림)
- 위 구조를 `ContiPrintLayout.tsx`(이미지 악보)와 `PdfSheetViewerClient.tsx`(PDF 악보) 양쪽 동일하게 적용

### 4. 찬양 검색 페이지 개선

- **악보 보기**: 곡 상세 모달(`SongsDetailModal`)에 악보 보기 섹션 추가
  - PDF: 새 탭 열기 버튼
  - 이미지: 모달 내 인라인 표시
- **악보 업로드 관리자 전용 제한**
  - `SongsFormModal`에 `isAdmin` prop 추가 → 비관리자는 악보 업로드 UI 숨기고 안내 문구 표시
  - **서버에도 동일하게 적용**: `POST /songs/:id/sheet`, `DELETE /songs/:id/sheet` 엔드포인트에 role 체크 추가 → 비관리자 접근 시 `403 Forbidden`

### 5. 콘티 편집 패널에서 새 찬양 직접 등록

- `AddSongPanel` 헤더에 "새 찬양 등록" 버튼 추가
- 검색 결과가 없을 때 `"[검색어] 등록하기"` 버튼 표시
- 클릭 시 `SongsFormModal`을 재사용하여 곡 등록 폼 오픈
- 등록 완료 후 목록 갱신 + 방금 만든 곡 자동 선택

## 버그픽스

### iOS 흰 화면

- **원인**: `firebase_core ^3.x`부터 `Firebase.initializeApp()` 단독 호출 불가
  - `GoogleService-Info.plist`가 있어도 options 명시 필요
- **수정**: `apps/lib/firebase_options.dart` 생성 후 `main.dart`에서 명시적 전달

  ```dart
  await Firebase.initializeApp(
    options: DefaultFirebaseOptions.currentPlatform,
  );
  ```

### iOS / Android GoRouter 흰 화면 (splash 멈춤)

- **원인**: GoRouter redirect에 unauthenticated 처리 누락
  - `_checkAuth()` 완료 후 토큰 없음 → unauthenticated 상태인데 `/login`으로 redirect하는 로직이 없어 `/splash`에서 멈춤
- **수정**: `apps/lib/main.dart` redirect에 추가

  ```dart
  if (!isAuth && !isAuthRoute) return '/login';
  ```

### 웹 CORS 에러

- **원인**: 서버 `CLIENT_URL`에 새 도메인(`https://worshiplog.inho.pe.kr`) 미포함
- **수정**: Oracle Cloud 서버 `.env`에서 `CLIENT_URL` 업데이트

  ```env
  CLIENT_URL=https://worshiplog.inho.pe.kr,http://localhost:3001
  ```

- **로컬 `.env`도 동일하게 업데이트** — 이후 서버에 붙여넣을 때 자동 반영되도록
